## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

### `src/`
This folder contains the main application code. Inside this directory, you will find the following subfolders:

- **`app.module.ts`**: The root module for your application. It registers and imports all other modules and services required by the app.

- **`modules/`**: Contains different feature modules (e.g., `users`, `auth`, `files`). Each module encapsulates a related set of components (controllers, services, entities).

  - **Module (`*.module.ts`)**: A module is a class annotated with `@Module` decorator. It serves as an organizational boundary within the application, grouping related controllers, services, and other providers together. Modules make it easy to break the app into isolated pieces.
    - Example: `UserModule`, `FileModule`.
  
  - **Controller (`*.controller.ts`)**: A controller is responsible for handling incoming HTTP requests and returning responses to the client. It defines API endpoints and typically delegates business logic to services.
    - Example: `UserController` will handle routes like `/users` and perform CRUD operations on user data.

  - **Service (`*.service.ts`)**: A service contains the business logic of the application. It is responsible for interacting with databases (using repositories), executing logic, and returning results to controllers.
    - Example: `UserService` will handle database interactions such as creating, fetching, updating, and deleting users.

  - **Entity (`*.entity.ts`)**: An entity represents a table in the database. It is a class decorated with `@Entity` (when using TypeORM) and defines the structure of the database table, including columns and relations with other tables.
    - Example: `UserEntity` represents a table with fields like `id`, `name`, `email`.

### `config/`
This folder contains configuration files like environment settings or database connection configurations.

### `test/`
Houses unit and end-to-end (e2e) test files to verify the functionality of your application.

## Installation

```bash
$ npm install
```

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
